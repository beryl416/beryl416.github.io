앱을 만들면 사람들이 쓸 수 있게 **서버**라는 걸 켜놔야 해. 나도 아무것도 모르는 상태에서 하나하나 따라 하면서 서버를 만들었는데, 그 과정에서 나온 개념들을 정리해봤어.

## 서버가 뭐야?

컴퓨터야. 그냥 컴퓨터. 다만 **24시간 켜져 있는 컴퓨터**야.

우리가 카카오톡으로 메시지를 보내면, 그 메시지가 어딘가에 있는 컴퓨터를 거쳐서 친구한테 전달돼. 그 "어딘가에 있는 컴퓨터"가 바로 서버야.

집에 있는 컴퓨터도 서버가 될 수 있어. 라즈베리파이 같은 작은 컴퓨터도 가능해. 하지만 집 컴퓨터는 정전되면 꺼지고, 인터넷이 끊기면 연결이 안 돼. 그래서 전문 업체의 서버를 빌려 쓰는 거야.

## VPS — 서버를 빌려 쓰는 것

**VPS(Virtual Private Server)**는 "가상 개인 서버"야.

비유하면 이래. 거대한 아파트(진짜 큰 서버 컴퓨터)가 있어. 그 아파트를 방 하나씩 나눠서 빌려주는 거야. 내 방은 다른 사람이 못 들어와. 나만의 공간이야. 하지만 실제로는 한 건물 안에 있지.

AWS Lightsail, Vultr, DigitalOcean 같은 서비스가 이런 방을 빌려줘. 월세처럼 매달 돈을 내고 쓰는 거야.

## 리전 — 서버가 있는 도시

**리전(Region)**은 서버가 **물리적으로 어디에 있느냐**야.

서울 리전을 고르면, 진짜로 서울에 있는 데이터센터에 내 서버가 만들어져. 미국 리전을 고르면 미국에 만들어지고.

왜 중요하냐면, **가까울수록 빨라**. 편지를 옆집에 보내는 거랑 미국에 보내는 거랑, 어느 게 빨리 도착할까? 당연히 옆집이지. 인터넷도 마찬가지야.

한국 사용자가 많으면 → 서울 리전
미국 사용자가 많으면 → 미국 리전

## IP 주소 — 서버의 집 주소

모든 컴퓨터는 인터넷에서 **주소**가 있어. 이걸 **IP 주소**라고 해. `3.36.19.0`처럼 숫자로 된 주소야.

친구 집에 놀러가려면 주소를 알아야 하잖아? 서버도 마찬가지야. 앱이 서버에 접속하려면 IP 주소를 알아야 해.

### 고정 IP vs 유동 IP

집에서 쓰는 인터넷은 보통 **유동 IP**야. 공유기를 껐다 켜면 주소가 바뀔 수 있어. 마치 이사를 계속 하는 것처럼. 그러면 친구가 찾아올 수가 없지.

서버는 **고정 IP**를 써. 주소가 절대 안 바뀌는 거야. 그래야 앱이 항상 같은 주소로 찾아올 수 있으니까.

## 포트 — 서버의 문 번호

IP 주소가 건물 주소라면, **포트(Port)**는 **문 번호**야.

한 서버에서 여러 프로그램이 돌아갈 수 있어. 웹사이트도 돌리고, 채팅 서버도 돌리고. 이때 "어느 프로그램한테 말할 건지"를 정하는 게 포트야.

자주 쓰는 포트 번호:
- **22번** — SSH (서버에 접속하는 문)
- **80번** — HTTP (웹사이트)
- **443번** — HTTPS (보안 웹사이트)

## 방화벽 — 문을 열고 닫는 것

**방화벽(Firewall)**은 어떤 문을 열어둘지, 닫아둘지 정하는 거야.

기본적으로 서버의 모든 문은 잠겨 있어. 보안 때문에. 필요한 문만 열어두는 거야.

우리가 Lightsail에서 한 건:
- **22번 문 열기** → 내가 서버에 접속할 수 있게
- **80번 문 열기** → 사람들이 웹사이트에 접속할 수 있게
- **443번 문 열기** → 보안 연결로 접속할 수 있게

안 쓰는 문은 닫아두는 게 좋아. 도둑이 들어올 수 있으니까.

## SSH — 서버에 원격 접속하기

**SSH(Secure Shell)**는 **멀리 있는 서버에 접속하는 방법**이야.

서버는 서울 데이터센터에 있는데, 나는 집에 있잖아. 직접 가서 키보드를 칠 수는 없으니까, 인터넷을 통해서 서버의 터미널(까만 화면)에 접속하는 거야. 마치 전화로 다른 사람한테 "이거 좀 타이핑해줘"라고 하는 것처럼. 다만 직접 타이핑하는 거지.

Lightsail에서는 브라우저에서 바로 SSH 접속이 가능해. "SSH를 사용하여 연결" 버튼을 누르면 까만 화면이 뜨고, 거기에 명령어를 입력하면 서버가 실행해.

## Node.js — 서버 프로그램을 돌리는 엔진

**Node.js**는 JavaScript라는 프로그래밍 언어로 만든 프로그램을 **서버에서 돌릴 수 있게 해주는 엔진**이야.

자동차에 엔진이 있어야 달릴 수 있는 것처럼, 서버 프로그램도 엔진이 있어야 돌아가. 우리 앱의 채팅 서버는 JavaScript로 만들었으니까, Node.js가 필요한 거야.

## nginx — 교통정리하는 경비원

**nginx(엔진엑스)**는 서버 앞에서 **교통정리를 해주는 프로그램**이야.

사용자가 서버에 접속하면, nginx가 먼저 받아서 "이건 웹사이트 요청이니까 여기로, 이건 채팅 요청이니까 저기로" 하고 안내해줘. 학교 앞 교통 지도 아저씨 같은 거야.

## SSL/HTTPS — 비밀 편지 봉투

인터넷으로 메시지를 보내면, 중간에 누가 엿볼 수 있어. 엽서처럼 내용이 다 보이는 거지.

**SSL**은 메시지를 **암호화**하는 거야. 엽서 대신 **봉투에 넣어서 봉인**하는 것처럼. 보내는 사람과 받는 사람만 내용을 볼 수 있어.

- **HTTP** = 엽서 (누구나 볼 수 있음)
- **HTTPS** = 봉투 편지 (나와 상대만 볼 수 있음)

주소창에 자물쇠 아이콘 🔒 보이면 HTTPS를 쓰고 있다는 뜻이야. 앱스토어에 앱을 올리려면 반드시 HTTPS를 써야 해.

## systemd — 서버 프로그램 자동 관리자

서버 프로그램이 갑자기 죽으면 어떡해? 새벽 3시에 서버가 멈추면 내가 일어나서 다시 켜야 해?

**systemd**가 이걸 자동으로 해줘. "이 프로그램이 죽으면 5초 후에 다시 켜라"라고 설정해두면, 알아서 되살려줘. 그리고 서버 컴퓨터가 재시작되면 프로그램도 자동으로 같이 켜져.

24시간 지켜보는 **자동 감시 로봇** 같은 거야.

## 스왑(Swap) — 임시 메모리

컴퓨터가 프로그램을 돌리려면 **메모리(RAM)**가 필요해. 책상이라고 생각하면 돼. 책상이 넓을수록 많은 걸 동시에 펼칠 수 있지.

그런데 우리 서버는 512MB밖에 안 돼. 책상이 좁은 거야. 프로그램을 설치하다가 "자리가 없어요!"하고 멈춰버렸어.

그래서 **스왑**을 만들었어. 하드디스크 일부를 임시 책상으로 쓰는 거야. 진짜 책상(RAM)보다는 느리지만, 없는 것보다 훨씬 나아.

## 도메인 — 예쁜 주소

`3.36.19.0` 같은 숫자 주소는 기억하기 어렵잖아. 그래서 **도메인**이라는 이름을 붙여줘.

- `3.36.19.0` → 숫자 주소 (기억하기 어려움)
- `blitz.app` → 도메인 (기억하기 쉬움)

전화번호부 같은 거야. "엄마"라고 검색하면 전화번호가 나오잖아. 인터넷에서는 도메인을 입력하면 IP 주소를 찾아서 연결해줘.

## 전체 흐름을 정리하면

사용자가 앱에서 메시지를 보내면 이런 일이 벌어져:

1. 앱이 **도메인**(예: blitz.app)으로 접속 시도
2. 도메인이 **IP 주소**(3.36.19.0)로 변환됨
3. **SSL/HTTPS**로 암호화된 연결이 만들어짐
4. **nginx**가 요청을 받아서 채팅 서버로 전달
5. **Node.js**에서 돌아가는 채팅 서버가 메시지를 처리
6. 상대방한테 메시지 전달

이 모든 게 **0.01초도 안 걸려**. 서울 안이라면.
